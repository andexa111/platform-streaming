1. Ringkasan Proyek

Platform Streaming Video adalah sistem berbasis subscription yang memungkinkan pengguna menonton konten film secara online.
Sistem dibangun dengan arsitektur monorepo menggunakan Next.js untuk frontend dan NestJS untuk backend.
Video streaming diamankan melalui Bunny.net CDN dengan Signed URL, dan pembayaran diproses via Midtrans.

2. Kebutuhan Fungsional

A. Autentikasi &amp; User Management

Fitur
Deskripsi
Registrasi
Daftar akun menggunakan email &amp; password, atau Google OAuth (OAuth 2.0).
Login &amp; Logout
Login dengan kredensial atau Google. Logout menghapus token di frontend.
Verifikasi Email
Token unik dikirim via Resend. Satu kali pakai, berlaku 24 jam.
Role-based Access
5 role: Guest, User, Subscriber, Admin, Super Admin. Dikontrol via JWT + guard.
Manajemen User
Admin dapat melihat, memfilter, dan mengubah status user (aktif/nonaktif).
Refresh Token
Endpoint untuk memperbarui JWT tanpa paksa login ulang.

B. Manajemen Konten Film

Fitur
Deskripsi
Upload Film
Admin upload file MP4 langsung ke Bunny (direct upload — tidak lewat server).
Upload Poster &amp; Trailer
Poster (jpg/png) dan trailer URL disimpan bersama metadata film.
Metadata Film
Judul, genre, sinopsis, durasi, tahun rilis, sutradara.
Jadwal Tayang
Admin set scheduled_at. Film otomatis published saat waktu tiba (via cron).
Coming Soon
H-7 sebelum tayang, poster tampil di beranda dengan label coming soon.
Sistem Rerun
Admin dapat menjadwalkan penayangan ulang film lama.
Edit &amp; Hapus Film
Edit metadata tanpa menyentuh Bunny. Hapus menggunakan soft delete (is_deleted).

C. Streaming Video

Fitur
Deskripsi
Integrasi Bunny.net
Video disimpan dan distream dari Bunny CDN. Backend hanya generate tiket akses.
Signed URL
Token SHA256 dari tokenSecret + path + expiry. Subscriber: 2 jam. User gratis: 25 menit.
Video Player
Player mendukung streaming adaptif, simpan posisi, dan tidak bisa di-inspect URL.
Akses Berbasis Role
Backend selalu query role dari DB (bukan JWT) sebelum generate signed URL.
Refresh URL Otomatis
Frontend request URL baru saat sisa 15 menit — seamless untuk film panjang.
Cek Jadwal Tayang
Film hanya bisa diakses jika is_published = true DAN scheduled_at sudah terlewati.

D. Subscription &amp; Payment

Fitur
Deskripsi
Pilihan Paket
Dua paket: bulanan dan tahunan. Harga dikonfigurasi di backend.
Integrasi Midtrans
Backend buat order ke Midtrans API, user dibayar di halaman Midtrans.
Aktivasi via Webhook
HANYA dari webhook Midtrans yang terverifikasi (SHA512 signature). Bukan dari redirect.
Idempotency Check
Cek apakah order sudah diproses sebelum aktivasi — antisipasi webhook duplikat.
DB Transaction
Update payments + subscriptions + users.role dalam satu atomic transaction.
Auto Expire
Cron job harian cabut akses subscriber yang sudah melewati expired_at.
Perpanjang Langganan
expired_at baru = expired_at lama + 30/365 hari (bukan dari now()).
Riwayat Pembayaran
User dapat melihat semua transaksi: tanggal, paket, status, nominal.

E. Sistem Akses Konten

Role
Hak Akses
Guest
Hanya bisa lihat daftar film, poster, jadwal tayang, dan trailer.
User Gratis
Nonton maksimal 20 menit. Menit ke-20: video pause, iklan muncul, lalu block.
Subscriber
Akses penuh semua film sesuai jadwal tayang. Tanpa iklan, tanpa batas waktu.

F. Sistem Iklan

Fitur
Deskripsi
Preload Data Iklan
GET /api/ads/active dipanggil saat player pertama load — tidak menunggu menit ke-20.
Trigger Iklan
Frontend timer 20 menit. Saat habis: player.pause() + simpan currentTime.
Pemutaran Iklan
Video iklan diputar. Countdown ditampilkan. Tidak bisa di-skip.
Block Setelah Iklan
Setelah iklan selesai, video film tetap di-block. Muncul modal subscribe.
Iklan Null
Jika tidak ada iklan aktif, langsung tampilkan modal subscribe tanpa crash.
Kelola Iklan (Admin)
Upload, aktifkan, nonaktifkan, hapus iklan. Hanya satu iklan aktif di satu waktu.

G. Notifikasi Email

Jenis Email
Keterangan
Verifikasi Email
Dikirim saat register. Token 24 jam, 1x pakai.
Konfirmasi Pembayaran
Dikirim setelah webhook subscriber aktif. Isi: paket, nominal, expired_at.
Reminder H-3
Dikirim 3 hari sebelum langganan habis. Flag reminder_sent mencegah duplikat.
Notif Expired
Dikirim saat cron job mencabut akses. Berisi link untuk perpanjang.
Notif Film Baru
Dikirim ke semua subscriber saat film baru di-publish (is_published = true).
Notif Rerun
Dikirim ke subscriber saat film lama dijadwalkan tayang ulang.

3. Kebutuhan Non-Fungsional

A. Performa
Aspek
Target
Streaming Video
Memanfaatkan CDN Bunny.net untuk latensi rendah. Video tidak melewati server.
Load Halaman
Target &lt; 2 detik untuk halaman beranda. Gunakan lazy loading untuk daftar film.
Skalabilitas
Backend stateless (JWT) — bisa di-scale horizontal tanpa sesi tersimpan di server.

B. Keamanan
Aspek
Implementasi
Enkripsi Password
bcrypt dengan cost factor minimum 12.
JWT Authentication
Token ditandatangani dengan secret key, disimpan di httpOnly cookie atau memory.
HTTPS Wajib
Semua komunikasi dienkripsi TLS. HTTP redirect ke HTTPS.
Validasi Webhook
SHA512 signature verification sebelum proses webhook Midtrans.
Signed URL Video
Token SHA256 dengan expiry. Bunny verifikasi setiap request stream.
Rate Limiting
Register &amp; login endpoint dibatasi untuk mencegah brute force.
Input Validation
Semua input divalidasi di backend (class-validator di NestJS).
MIME Type Check
Validasi MIME type file upload — tidak hanya cek ekstensi.

C. Reliability &amp; Maintenance
Aspek
Implementasi
Error Handling
Global exception filter di NestJS. Semua error dikembalikan dalam format konsisten.
Logging
Semua request, error, dan webhook dicatat (Winston atau Pino).
Idempotency
Webhook payment diproses maksimal satu kali meski dikirim berulang.
Backup Database
Backup MySQL terjadwal. Simpan minimal 7 hari terakhir.
Monitoring
Pantau uptime server dan error rate.

D. Infrastruktur
Komponen
Spesifikasi
VPS
Minimal 2 vCPU, 4GB RAM, 100GB SSD. Disarankan Ubuntu 22.04.
CDN Video
Bunny.net Storage Zone + Pull Zone dengan token authentication aktif.
Email Service
Resend — gratis 3.000 email/bulan. Verifikasi domain pengirim.
Payment Gateway
Midtrans — Sandbox untuk development, Production untuk live.
Database
MySQL 8.0 + Prisma ORM. Gunakan connection pooling.

7. Rencana Testing

Testing per Role User
Role
Skenario Test

Lihat beranda, lihat trailer, coba klik tonton (harus muncul prompt login), akses register.

Login, nonton film, tunggu menit ke-20, iklan muncul, modal subscribe muncul, video block.

Testing Kritis
Skenario
Cara Test
Payment Sandbox
Gunakan akun Midtrans Sandbox. Test sukses, gagal, dan cancel. Verifikasi webhook masuk dan subscriber aktif.
Webhook Duplicate
Kirim webhook yang sama dua kali. Verifikasi hanya satu aktivasi terjadi (idempotency).
Signed URL
Pastikan URL user gratis expired setelah 25 menit. Pastikan URL tidak bisa dipakai oleh role lain.
Cron Job
Set expired_at manual ke masa lalu. Jalankan cron. Verifikasi role berubah ke user dan email terkirim.
Concurrent User
Simulasi beberapa user nonton bersamaan. Pastikan server tidak overload.

Catatan Penting Testing:
  - Selalu gunakan Midtrans Sandbox (bukan Production) selama development.
  - Untuk test webhook lokal, gunakan ngrok untuk expose endpoint ke internet.
  - Simpan collection Postman/Insomnia untuk semua endpoint sebagai dokumentasi API.
  - Setelah testing selesai, buat checklist sign-off sebelum deploy ke production.
